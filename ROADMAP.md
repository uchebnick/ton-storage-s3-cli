# 🗺 Дорожная карта (Roadmap)

Этот план описывает будущее развитие TON Storage S3 Gateway.

## 🚧 Фаза 1: Стабильность и Надежность (Текущий фокус)
Цель: сделать систему устойчивой к сбоям сети и текучке провайдеров.

- [x] Базовая реализация S3 PUT/GET/DELETE.
- [x] Демон авто-репликации (Replicator).
- [x] Выгрузка локальных данных (Cleaner).
- [x] Мониторинг провайдеров (Auditor).
- [ ] **Комплексный набор тестов:**
    - [ ] Unit-тесты для логики `Replicator` (с моками TON интерфейсов).
    - [ ] Unit-тесты для SQL запросов `Cleaner` и `Auditor`.
    - [ ] Интеграционные тесты: Поднятие локальной `tonutils-storage` testnet ноды и проверка передачи файлов.
    - [ ] E2E тесты для S3 API (использование `minio-client` в CI/CD).
- [ ] **Graceful Shutdown:** Гарантировать, что задачи загрузки (`downloads`) корректно ставятся на паузу/сохраняются при SIGTERM.
- [ ] **Улучшенная обработка ошибок:** Кастомные типы ошибок для "Torrent not found", "Insufficient Balance".

## 🛠 Фаза 2: Расширенные возможности
Улучшение юзабилити и безопасности данных.

- [ ] **Шифрование (Encryption at Rest):** Шифрование файлов на стороне клиента перед созданием TON Bag.
- [ ] **Сжатие:** Gzip/Zstd сжатие для не-медиа файлов перед загрузкой.
- [ ] **Поддержка Multi-Wallet:** Использование разных кошельков для разных бакетов.



## 🚀 Фаза 3: Масштабирование и Enterprise
Готовность к высоконагруженным средам.

- [ ] **Кластерный режим:** Разделение `Gateway`, `Replicator` и `Downloader` на разные микросервисы.
- [ ] **Распределенная БД:** Поддержка CockroachDB или кластерного Postgres.
- [ ] **S3 Object Locking:** Неизменяемые бэкапы (WORM - Write Once Read Many).
- [ ] **Метрики Prometheus:** Экспорт детальных метрик (скорость загрузки, активные пиры, баланс TON).

## 💡 Долгосрочное видение
Стать стандартным open-source шлюзом для всех, кто хочет использовать TON Storage как бэкенд для традиционных Web2 приложений.
